# Object Formation

## За что отвечает стадия

Детектор движения обычно даёт маску, а не удобные объекты. Эта стадия превращает сырые пятна в список объектов с bbox и метриками.

## Что уже реализовано

Рабочие классы:
- `ConnectedComponentsObjectBuilder`
- `ContourObjectBuilder`

## Что подготовлено на будущее

Заглушки:
- `BlobObjectBuilder`
- `WatershedObjectBuilder`
- `DistanceTransformSplitObjectBuilder`

## Когда какой вариант полезен

- `ConnectedComponents` — самый прямой и понятный путь от маски к объектам.
- `Contour` — полезен, когда важна внешняя форма области.
- `Blob` — удобен для компактных целей.
- `Watershed` — нужен, когда цели слипаются.
- `DistanceTransformSplit` — хороший резерв для плотных скоплений.

## Практический вывод

Для будущего automatic mode эта стадия будет очень важной, особенно если нужно разлеплять близкие объекты.
