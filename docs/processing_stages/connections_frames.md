# Frame Source

## За что отвечает стадия

Эта стадия отвечает только за одно: откуда брать очередной кадр.

Она не должна знать ничего о трекинге, пресетах, bbox и тепловизоре. Её работа — честно выдавать кадры и закрываться без истерик.

## Что уже есть

Реально готовые классы:
- `OpenCvVideoSource` — чтение обычного видео через `cv2.VideoCapture`.
- `ImageSequenceFrameSource` — чтение кадров из папки с изображениями.
- `ReplayFrameSource` — воспроизведение заранее переданного списка кадров.

## Что подготовлено на будущее

Заглушки:
- `RtspStreamFrameSource` — для живого потока.
- `MultiCameraFrameSource` — для сценария с несколькими камерами.

## Когда что использовать

- `OpenCvVideoSource` — текущий основной режим.
- `ImageSequenceFrameSource` — удобно для отладки на покадровых наборах.
- `ReplayFrameSource` — удобно для тестов и повторяемых экспериментов.
- `Rtsp` и `MultiCamera` — точки расширения под реальную систему.
